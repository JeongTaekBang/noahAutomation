# NOAH Dashboard 사용 가이드

---

## 1. 소개

NOAH 인터컴퍼니 거래 현황을 한눈에 파악할 수 있는 대시보드입니다.
Excel로 관리 중인 수주(SO)/발주(PO)/출고(DN) 데이터를 자동으로 시각화합니다.

### 주요 기능 (8페이지)

| 페이지 | 설명 |
|--------|------|
| 오늘의 현황 | KPI 카드, 납기 캘린더, PO 확정 지연, EXW 완료 미출고, 납기 현황, 해외 선적 Action Items, 세금계산서 미발행 |
| 수주/출고 현황 | 월별 수주/출고 추이, Book-to-Bill, 일별 수주/출고 히트맵 |
| 제품 분석 | 매출 Top 15(클릭 드릴다운), 구성비, 월별 추이, 평균단가, Backlog, Deep Analysis(ABC/성장률/매트릭스) |
| 섹터 분석 | 섹터별 매출(클릭 드릴다운), 월별 추이, 제품 믹스, Backlog, 평균 주문 규모 |
| 고객 분석 | 고객별 매출 Top 15(클릭 드릴다운), Pareto, 월별 추이, Deep Analysis(ABC/성장률/수익성) |
| 발주 커버리지 | 미발주/부분발주/발주진행중/발주확정 KPI, 상세 테이블, 고객별/섹터별 분석 |
| 수익성 분석 | 매출 vs 원가(ICO) 월별 마진 추이, 3탭(고객별/섹터별/모델별), 저마진 경보 Top 10 |
| Order Book | Executive(워터폴/KPI), Risk(Aging/위험건), Conversion(리드타임/Box plot) |

## 2. 사용 방법

**별도 프로그램 설치가 필요 없습니다.**

### 실행

1. 공유받은 **NOAH_Dashboard** 폴더를 본인 PC에 복사합니다.
2. **NOAH Dashboard.bat** 파일을 더블클릭합니다.
3. 잠시 후 브라우저에 대시보드가 자동으로 열립니다.
4. 종료 시 검은 창(콘솔)을 닫으면 됩니다.

### DB 파일

- `noah_data.db`는 OneDrive를 통해 공유되며, 대시보드가 자동으로 인식합니다.
- 자동 인식이 안 되는 경우: `dashboard_config.ini` 파일을 열어 DB 경로를 직접 입력합니다.

```ini
[database]
db_path = C:\Users\홍길동\OneDrive - Rotork plc\...\noah_data.db
```

### 사이드바 필터

모든 페이지에 공통 필터가 적용됩니다:

- **시장 구분**: 전체 / 국내 / 해외
- **연도**: 전체 또는 특정 연도 선택
- **월**: 전체 또는 특정 월 선택 (연도 = 전체 시 비활성)
- **섹터**: 복수 선택 가능
- **고객**: 접이식(Expander) 내 복수 선택 가능
- **데이터 새로고침**: 캐시 초기화 (5분마다 자동 갱신)
- **마지막 동기화**: DB 마지막 sync 시각 표시
- **데이터 로드 오류 배너**: DB 쿼리 실패 시 페이지 상단에 경고 표시 (어떤 로더가 실패했는지, 에러 유형 포함). "데이터 없음"과 "쿼리 실패"를 구분 가능

## 3. 페이지별 안내

### 오늘의 현황

당일 기준 주요 Action Items:

- **KPI 카드 (4개)**: 오늘 납기 / 이번 주 납기 / 금월 수주 / 금월 출고
- **납기 캘린더**: 월별 히트맵, 날짜 클릭 시 상세 드릴다운
  - EXW 출고 예정 / 공장 픽업 / 납기 예정 / 출고 실적
- **PO 확정 지연**: Sent → Confirmed 대기 건, Aging 버킷별(7일/8~14일/15~30일/30일+) 분류
  - 국내 / 해외 탭 분리
- **EXW 완료 미출고**: PO EXW 일자 경과 but 미출고(Invoiced 아님) 건
  - 라인 아이템 단위, Aging 버킷별 분류, 국내 / 해외 탭 분리
- **납기 현황**: 납기일 경과 but DN 미완료/부분 출고 건
  - 잔여 수량/금액 표시, 국내 / 해외 탭 분리
- **해외 선적 Action Items** (시장 ≠ 국내 시 표시):
  - 선적 대기 / 포워더 미정 건
  - 공장 출고일, 픽업일, 선적 예정일, B/L, 포워더 정보
- **세금계산서 미발행** (국내 전용):
  - KPI 4개(건수/금액/최대 경과일/30일 초과 건수)
  - Aging 바 차트 + 고객별 금액 Top 10

### 수주/출고 현황

- **KPI 카드 (6개)**: 금월 수주(전월 대비) / 전월 수주 / 누적 수주 / 금월 출고(전월 대비) / 전월 출고 / Book-to-Bill
- **월별 수주/출고 금액 추이**: 그룹 막대 + 누적매출 라인
- **Book-to-Bill 월별 추이**: 라인 차트 (1.0 기준선)
- **일별 수주/출고 현황**: 월 선택기 + SO/DN 듀얼 히트맵

### 제품 분석

- **KPI 카드 (3개)**: 총 제품 종류 / 최다 판매 제품 / 최고 매출 제품
- **제품별 매출 Top 15**: 가로 막대, 클릭 시 드릴다운 (월별 추이, 섹터 비중, 주요 고객 Top 5)
- **제품 구성비**: 도넛 차트 (Top 8 + 기타)
- **제품별 월별 추이 (Top 5)**: 누적 영역 차트
- **제품별 평균 단가 Top 15**: 가로 막대
- **제품별 Backlog Top 10**: 가로 막대
- **Deep Analysis**:
  - 제품 세그먼트 (ABC): 80/95 기준 3등급 분류
  - 제품 성장률: 전월 대비 성장 Top 5 / 하락 Top 5
  - 제품 × 고객 매트릭스: 히트맵 (Top 10 제품 × Top 10 고객)

### 섹터 분석

- **KPI 카드**: 섹터별 매출 비율 (최대 6개, 금액 + 비중%)
- **섹터별 매출 비율**: 파이 차트, 클릭 시 드릴다운 (제품 믹스, 월별 추이, 주요 고객 Top 5)
- **섹터별 월별 추이**: 누적 막대 차트
- **섹터별 제품 믹스**: 그룹 막대 (Top 8 제품)
- **섹터별 Backlog 현황**: 건수 + 금액 테이블
- **섹터별 평균 주문 규모**: 막대 차트

### 고객 분석

- **KPI 카드 (3개)**: 총 고객 수 / Top 고객 / 금월 수주 고객 수
- **고객별 매출 Top 15**: 가로 막대, 클릭 시 드릴다운 (월별 추이, 제품 믹스 도넛, Backlog 상세)
- **고객 집중도 (Pareto)**: 막대 + 누적 라인 (Top 20)
- **고객 상세 테이블**: 주문 건수, 수량, 금액, 최근 월, 평균 주문, Backlog
- **고객별 월별 매출 추이 (Top 5)**: 마커 라인 차트
- **Deep Analysis**:
  - 고객 세그먼트 (ABC): 3등급 분류
  - 고객 성장률: 전월 대비 성장 Top 5 / 하락 Top 5
  - 고객 수익성: 고객별 마진율 분석

### 발주 커버리지

PO Status 기반 발주 현황:

| 상태 | 의미 |
|------|------|
| 미발주 | PO 없음 또는 Open (공장 발주 전) |
| 부분 발주 | 일부 Open + 일부 Sent/Confirmed |
| 발주 진행중 | Sent (공장에 발주, 확인 대기) |
| 발주 확정 | Confirmed / Invoiced |

- **KPI 카드 (5개)**: 미발주 / 부분 발주 / 발주 진행중 / 발주 확정 / 발주 필요 금액
- **커버리지 요약**: 상태별 누적 가로 막대 (컬러 구분)
- **상태별 상세 테이블**: 미발주 / 부분 발주 / 발주 진행중 각각 SO별 상세
- **고객별 발주필요 금액 Top 10**: 가로 막대
- **섹터별 커버리지율**: 막대 차트 (100% 기준선)
- 출고 완료 건, PO 전체 Cancelled 건은 분석에서 자동 제외

### 수익성 분석

SO 매출 vs PO 원가(ICO) 마진:

- **KPI 카드 (4개)**: 총매출 / 총원가(ICO) / 총마진 / 마진율(%)
- **월별 마진 추이**: 그룹 막대(매출/원가) + 마진율 라인
- **3개 탭** (고객별 / 섹터별 / 모델별):
  - 각 탭: 마진율 Top 15 막대 차트 + 상세 테이블(매출, 원가, 마진, 미출고금액)
- **저마진 경보 Top 10**: 마진율 20% 미만 건
- 원가 미확정 건(PO 없음)은 마진 계산에서 제외되며 별도 안내

### Order Book (3탭)

- **Executive**:
  - 워터폴 차트 (월별 선택 / 누적 토글): Opening → Input → Output → Ending
  - KPI 3개: Backlog Cover(월), Past Due Ratio(%), Book-to-Bill
  - 월별 Order Book 테이블
- **Risk**:
  - 납기 지연 + 임박 건 그룹 분석 (고객, 시장, 섹터)
  - Aging 버킷 시각화
- **Conversion**:
  - Output → 매출 전환 리드타임 분석
  - 해외 물류 리드타임 (공장 → 픽업 → 선적): Box plot + 통계 요약 + 이상치 감지

## 4. 향후 계획 — ERP 데이터 활용

현재는 NOAH 데이터만 대상이지만, 동일한 방식으로 ERP 데이터 활용이 가능합니다.

```
ERP (D365 / AX2009) → Excel/CSV 다운로드 → DB 동기화 → Dashboard
```

확장 가능 영역:

| 데이터 | 활용 |
|--------|------|
| Trial Balance (TB) | P&L, BS 월별 추이 대시보드 |
| AP/AR Aging | 미수금/미지급 현황 |
| 재고 데이터 | 재고 회전율, 과잉재고 분석 |

ERP에서 Excel/CSV로 데이터를 내려받으면, 원하는 형태의 분석 대시보드를 만들 수 있습니다.

## 5. 문제 해결

| 증상 | 해결 |
|------|------|
| bat 실행 시 바로 닫힘 | 콘솔 메시지 확인 — DB 경로 또는 파일 누락 |
| 빈 화면 (데이터 없음) | 데이터 새로고침 버튼 클릭 / DB 파일 경로 확인 |
| 포트 충돌 | Streamlit이 자동으로 빈 포트를 찾으므로 보통 문제 없음 |
| 상단에 경고 배너 표시 | 일부 데이터 로더가 실패한 상태. 배너에 표시된 테이블/에러 확인 후 `python sync_db.py` 재실행 또는 DB 스키마 점검 |
| DB 자동 인식 안 됨 | `dashboard_config.ini`에 경로 직접 입력 |
