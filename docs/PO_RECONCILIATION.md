# PO 매입대사 (reconcile_po.py)

> 운영자 관점 시각 가이드는 [매입대사_가이드.md](매입대사_가이드.md) 참고. 본 문서는 기술 참고용.

## 목적

NOAH 공장에서 출고된 금액과 회계팀이 AX에 입력한 GRN(Goods Received Note) 금액이 AX PO 번호 기준으로 일치하는지 확인한다.

## 사용법

```bash
python reconcile_po.py P03        # 3월 대사
python reconcile_po.py P03 -v     # 상세 로그
```

bat 메뉴에서 `[R]` 선택 후 월 코드 입력으로도 실행 가능.

## 데이터 소스 (3개)

| 소스 | 파일 위치 | 조인 키 | 금액 컬럼 | 비고 |
|------|----------|---------|----------|------|
| 회계 GRN | `po_reconciliation/2026/P03/Noah purchase_GRN_P03_2026.xlsx` | Purchase order (P######) | Cost amount physical | 회계팀 제공, **대사 기준** |
| 공장 출고 | `po_reconciliation/2026/P03/2026리스트_RCK_P03.xlsx` → Delivery 시트 | RCK ODER (ND-xxxx 또는 P######) | 계산서금액 | 공장 전체 출고 내역 |
| 내부 PO | `NOAH_SO_PO_DN.xlsx` → PO_국내/PO_해외 시트 | PO_ID ↔ AX PO | Total ICO | 담당자가 관리하는 오더만 |

### 파일 구조

```
po_reconciliation/
  2026/
    P03/
      Noah purchase_GRN_P03_2026.xlsx    ← 회계 GRN
      2026리스트_RCK_P03.xlsx            ← 공장 출고
      대사결과_P03.xlsx                   ← 출력
```

연도/월별로 폴더를 만들고 해당 월 파일을 넣는다. 스크립트는 월 코드(P03)로 폴더와 파일을 자동 매칭하며, 연도 폴더가 없는 평면 구조(`po_reconciliation/P03/`)도 호환된다.

**핵심 관계**

```
하나의 주문(PO_ID)이 여러 AX PO를 가질 수 있다 (1:N)

  ND-0214 ──┬── P023006 (Total ICO: 176,000)
            └── P023007 (Total ICO: 479,235)

  출고리스트에는 ND-0214 하나로 합산 (계산서금액: 655,235)
  GRN에는 P023006, P023007 각각 별도로 존재
  → 대사는 AX PO(P번호) 단위로 비교
```

**비교 방식**

```
  Product(국내/해외)     Service/YTC
  ──────────────────     ────────────────
  PO 금액 vs GRN 금액    출고 금액 vs GRN 금액
  (Total ICO)            (계산서금액)
```

## 처리 흐름

```
1. 파일 찾기
   po_reconciliation/{연도}/{월코드}/ (또는 평면 {월코드}/) 에서 GRN, 출고리스트 파일 자동 탐색

2. 데이터 로드
   ├─ 내부 PO: PO_국내 + PO_해외 (PO_ID ~ Status), Cancelled 제외
   ├─ 출고 리스트: Delivery 시트
   └─ GRN: Purchase order 기준

3. AX PO 통일 (조인 키 맞추기)
   ├─ ND-xxxx/NO-xxxx → PO시트에서 AX PO 매핑 (1:N 지원, 행 복제)
   └─ P###### → 그대로 AX PO

4. 내부 PO 필터링
   └─ Status = "Invoiced {월코드}" 인 건만 비교 대상

5. 매칭 판정 (GRN 기준)
   └─ 각 AX PO에 대해 PO/출고/GRN 존재 여부 + 금액 비교

6. Excel 출력
```

## 매칭 판정 로직

GRN의 각 AX PO에 대해 순서대로 판정:

```
GRN에 있는 AX PO
  ├─ 내부 PO(Invoiced)에 있음 → PO 금액 vs GRN 금액 비교
  │   ├─ 국내: 일치 / 불일치
  │   └─ 해외: 일치(해외) / 불일치(해외)
  │
  ├─ 내부 PO에 없고, 출고 리스트에 있음
  │   ├─ Type = Service/YTC → 출고 금액 vs GRN 금액 비교
  │   │   └─ 일치(서비스) / 불일치(서비스)
  │   └─ Type = Product → AX PO 누락 (PO시트에 AX PO 미입력)
  │
  └─ 내부 PO에도, 출고 리스트에도 없음 → 서비스/기타
```

### 매칭상태 일람

| 매칭상태 | GRN | 출고 | PO | 비교 기준 | 설명 |
|---------|-----|------|-----|----------|------|
| 일치 | O | - | O(국내) | PO = GRN | 국내 PO 금액과 GRN 일치 |
| 불일치 | O | - | O(국내) | PO ≠ GRN | 국내 PO 금액과 GRN 불일치 |
| 일치(해외) | O | - | O(해외) | PO = GRN | 해외 PO 금액과 GRN 일치 |
| 불일치(해외) | O | - | O(해외) | PO ≠ GRN | 해외 PO 금액과 GRN 불일치 |
| 일치(서비스) | O | O | X | 출고 = GRN | 서비스/YTC, 출고 금액과 GRN 일치 |
| 불일치(서비스) | O | O | X | 출고 ≠ GRN | 서비스/YTC, 출고 금액과 GRN 불일치 |
| AX PO 누락 | O | O | X | - | Product인데 PO시트에 AX PO 미입력 |
| 서비스/기타 | O | X | X | - | GRN에만 존재 (출고·PO 둘 다 없음) |
| GRN 미포함 | X | O/- | O/- | - | 출고/PO에 있지만 당월 GRN에 없음 |

## 출력 파일 (대사결과_P03.xlsx)

| 시트 | 내용 |
|------|------|
| **대사** | AX PO별 1행 요약: AX PO, 비교_금액, GRN_금액, 매칭상태 |
| **GRN_미포함** | GRN에 없는 AX PO 목록: AX PO, PO_금액 |
| raw_data | 소스별 상세 (long format): AX PO당 GRN/출고/PO 각 1행 |
| raw_data_미포함 | GRN 미포함 건의 소스별 상세 |
| AX_PO_미입력 | Invoiced인데 AX PO가 비어있는 PO 목록 (PO_ID, SO_ID 표시) |
| 범례 | 매칭상태 설명표 |

### 대사 시트 (요약)

```
AX PO    | 비교_금액  | GRN_금액   | 매칭상태
P022562  | 400,800   | 400,800   | 일치
P022939  | 160,000   | 160,000   | 일치(서비스)
P023006  | 176,000   | 176,000   | 일치
```

- 비교_금액: PO 금액 (국내/해외) 또는 출고 금액 (서비스)
- 매칭상태가 한눈에 확인 가능

### raw_data 시트 (상세)

```
AX PO    | 소스                        | 건수 | 금액
P022562  | Noah purchase_GRN_P03_2026  |    3 | 400,800   ← GRN (항상 첫 행)
P022562  | 2026리스트_RCK_P03          |    1 | 400,800   ← 출고
P022562  | NOAH_SO_PO_DN               |    3 | 400,800   ← PO
```

- GRN 행이 각 AX PO 그룹의 첫 번째
- 매칭상태는 GRN 행에만 표시
- 소스 필터로 특정 파일 데이터만 확인 가능
- 피벗: 행=AX PO, 열=소스, 값=금액 → 비교

## 주요 설계 결정

### 1:N 매핑 (PO_ID → AX PO)
하나의 PO_ID(ND-xxxx)가 여러 AX PO에 매핑될 수 있다. 예: ND-0214 → P023006, P023007. 이 경우 출고 리스트의 해당 행이 AX PO 수만큼 복제된다. 출고 금액은 합산되지만, 매칭 판정은 PO vs GRN으로 하기 때문에 AX PO별로 정확하게 비교된다.

### Cancelled 제외
PO시트의 Status가 Cancelled인 건은 매핑과 비교 모두에서 제외.

### Invoiced 필터
PO 비교 대상은 해당 월의 Invoiced 건만 (예: `Invoiced P03`). 매핑용 PO_ID → AX PO는 전체 PO에서 생성 (Delivery 변환에 필요).

### 해외 오더
해외(PO_해외)는 공장 출고 리스트에 별도 집계되지 않으므로 PO vs GRN으로만 비교한다. 매칭상태에 `(해외)` 표시.

### 서비스/YTC
PO시트를 거치지 않는 오더. 출고 리스트의 Type이 Service 또는 YTC인 경우 출고 금액(계산서금액) vs GRN으로 비교한다.
